Understanding Certificate File Formats

A comprehensive guide on PEM, DER, PFX/P12, and JKS encodings and how to convert between them securely.

When working with SSL/TLS and PKI infrastructure, dealing with various certificate file extensions can be confusing. The core difference usually boils down to whether the data is Base64 encoded (ASCII) or strictly Binary.

Format Comparison Overview

PEM (Privacy Enhanced Mail)

  • Encoding: Base64 (ASCII)
  • Extensions: .pem, .crt, .cer, .key
  • Contains: Single cert or key
  • Readable: Yes (open in Notepad)

PFX / P12 (PKCS#12)

  • Encoding: Binary (Encrypted)
  • Extensions: .pfx, .p12
  • Contains: Cert + PrivKey + Chain
  • Readable: No (requires password)

OpenSSL Conversion Paths

Use the following commands to convert between formats. Make sure you have OpenSSL installed.

Convert PEM to DER

openssl x509 -outform der -in cert.pem -out cert.der

Extract Public Key from PFX

You will be prompted for the import password.

openssl pkcs12 -in bundle.pfx -clcerts -nokeys -out cert.pem

Common Errors & Troubleshooting

If you encounter issues loading certificates into a load balancer or web server, refer to these common errors.

error:0906D06C:PEM routines:PEM_read_bio:no start line

Cause: OpenSSL expected a PEM format but the file is binary (DER) or corrupted.

Fix: Convert DER to PEM using: `openssl x509 -inform DER -in cert.der -out cert.pem`
